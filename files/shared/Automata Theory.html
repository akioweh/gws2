Automata Theory

In Automata Theory, an Automaton is
a self-propelled computing device that follows predetermined (self-contained) instructions automatically on input.

As the point of Automata Theory is to model real-life computation machines, their formal definitions vary by author according to what exactly the author is trying to model.
Below is an overview of a basic, generic definition of an automaton which then specializes to those generally used to study Formal Languages & Grammar.

An Automaton is a 5-tuple M=⟨Σ,Γ,Q,δ,λ⟩M = \braket{\Sigma, \Gamma, Q, \delta, \lambda}M=⟨Σ,Γ,Q,δ,λ⟩:

  • Σ\SigmaΣ – Input Alphabet
    a finite set of symbols
  • Γ\GammaΓ – Output Alphabet
    a finite set of symbols
  • QQQ – States
    a set of states
  • δ\deltaδ – Transition Function
    δ:Q×Σ↦Q\delta : Q \times \Sigma \mapsto Qδ:Q×Σ↦Q
  • λ\lambdaλ – Output Function
    λ:Q×Σ↦Γ\lambda: Q \times \Sigma \mapsto \Gammaλ:Q×Σ↦Γ

An automaton is finite when QQQ is finite.
λ\lambdaλ and Γ\GammaΓ are “optional”; an automaton is not required to “output symbols”.

Automata can be nicely visualized (and represented) as transition diagrams – directed graphs whose vertices are states and edges are transitions (labelled with the necessary symbol(s) to “traverse” them).

Input

An input (word) www is a string of symbols a1a2a3…ana_1 a_2 a_3 \dots a_na1​a2​a3​…an​ for an∈Σa_n \in \Sigmaan​∈Σ. In Kleene Star notation, w∈Σ∗w \in \Sigma^\astw∈Σ∗.
An automaton reads an input word one symbol at a time.

Run

A run is a sequence of states q0q1q2…qnq_0 q_1 q_2 \dots q_nq0​q1​q2​…qn​ where qn∈Qq_n \in Qqn​∈Q and qi+1=δ(qi,ai+1)q_{i+1} = \delta(q_i, a_{i+1})qi+1​=δ(qi​,ai+1​).

I.e., an automaton starts at state q0q_0q0​, then for each input symbol, advances one state according to the transition function δ\deltaδ. After reading the entire input word, the automaton’s run ends on state qnq_nqn​, the final state.

Should the automaton have a λ\lambdaλ, it also emits an output symbol b∈Γb \in \Gammab∈Γ for each transition where bi+1=λ(qi,ai+1)b_{i+1} = \lambda(q_i, a_{i + 1})bi+1​=λ(qi​,ai+1​).

The inductive extension of δ\deltaδ into δˉ:Q×Σ∗↦Q\bar \delta : Q \times \Sigma^\ast \mapsto Qδˉ:Q×Σ∗↦Q maps entire input words to final states.

δˉ(q,w)≔{qw=εδ(δˉ(q,u),a)w=ua,a∈Σ\bar \delta(q, w) \coloneqq \begin{cases} q & w = \varepsilon \\ \delta\big(\bar\delta(q, u), a\big) & w = ua, a \in \Sigma \end{cases} \\[-1.5em]δˉ(q,w):={qδ(δˉ(q,u),a)​w=εw=ua,a∈Σ​
where ε\varepsilonε is the empty string.
λˉ\bar\lambdaλˉ can be constructed in the same way.

This abstracts away the “process of computation” and models automata as input-output functions.
For automata that do not have a λ\lambdaλ (i.e., that do not output symbols), their “output” is taken as qnq_nqn​, the final state.

Acceptor

To ship the two lovebirds automata and Formal Languages, we equip an (λ\lambdaλ, Γ\GammaΓ-less) automaton with

  • q0∈Qq_0 \in Qq0​∈Q, a distinguished state, the start state; and,
  • F⊆QF \subseteq QF⊆Q, a distinguished set of states, the accepting states,

to form an acceptor (the variant of automata used to study formal languages, and everything discussed hereafter).

Observe that acceptors are boolean-output functions.

Accepting Word
A word w∈Σ∗w \in \Sigma^\astw∈Σ∗ is an accepting word for acceptor MMM if δˉM(q0,w)∈FM\bar\delta^M(q_0, w) \in F^MδˉM(q0​,w)∈FM.
Recognized Language
The language recognized by (or the language of) an acceptor is the set of all its accepting words. I.e.,
MMM recognizes L⊆Σ∗L \subseteq \Sigma^\astL⊆Σ∗ iff L={w∈ΣM∗∣δˉM(q0M,w)∈FM}L = \big\{ w \in \Sigma^{M\ast} \mid \bar\delta^M(q_0^M, w) \in F^M \big\}L={w∈ΣM∗∣δˉM(q0M​,w)∈FM}.
I.e., MMM will tell you for all w∈Σ∗w \in \Sigma^\astw∈Σ∗ if w∈Lw \in Lw∈L – if www is a sentential form.
Recognizable Languages
The (class of) recognizable languages for a class of automata is, well, the set of languages each of which is recognized by some automata of said class.
(See below for two automata classes: Finite and Pushdown.)

In addition, automata are commonly modeled with forms of memory (e.g., pushdown automata, below) and non-functional transitions (one-to-many, aka. nondeterminism).
Nondeterministic automata can be in more than one state at once, due to transitions with multiple output states; i.e., δ\deltaδ is a binary relation, and no longer a function.

Note that nondeterminism here solely refers to the ability for an automaton to be in more than one state at once; it does not mean the automaton produces nondeterministic results (runs are always deterministic).

Finite Automata

Aka. Finite State Machine (FSM).

A Finite Automaton (FA) is a 5-tuple ⟨Q,Σ,δ,q0,F⟩\braket{Q, \Sigma, \delta, q_0, F}⟨Q,Σ,δ,q0​,F⟩ (see acceptor above):

  • QQQ – States
  • Σ\SigmaΣ – Alphabet
  • δ\deltaδ – Transition Function
  • q0q_0q0​ – Initial State
  • FFF – Accepting States

Note that FA has no memory / auxiliary storage. Transitions depend purely on current state and input symbol.

Nondeterministic FA (NFA) and Deterministic FA (DFA) are equivalent (always inter-convertible).

Recognizes all Regular (Type-3) Languages.
LLL is regular iff there exists a DFA whose langauge is LLL.

Completeness:
A DFA is complete if δ\deltaδ is total.
Technically all DFAs should be complete, implying, in practice, that we explicitly define “dead”/“trash” states… but if not, undefined transitions are assumed to denote failure (immediate rejection), or the transition into an implicit non-accepting dead state.

Nondeterministic Finite Automaton

  • δ\deltaδ is a binary relation (it can map one-to-many states for the same input symbol) (or as δ:Q×Σ↦P(Q)\delta : Q \times \Sigma \mapsto \mathcal{P}(Q)δ:Q×Σ↦P(Q))
  • ε\varepsilonε-transitions are allowed (transitions may consume no symbols)
  • no explicit “dead” state needed; NFAs need not be complete

ε\bm\varepsilonε-Closure
As we now have ε\varepsilonε-transitions, for any current state, all other states accessible via ε\varepsilonε-transition(s) are also current-states.
The set of ε\varepsilonε-reachable states from a given state (which includes itself) is its ε\varepsilonε-closure.
I.e., the transitive closure of undefinedε\xrightarrow{\varepsilon}ε​.

Pumping Lemma

All sufficiently long sentences (strings) of a regular language can be pumped – have a middle substring repeated (self-concat) an arbitrary number of times while still remaining within the language.
“Sufficiently long” is defined by a length threshold ppp.

Finite languages vacuously satisfy the lemma (take p>p >p> longest sentence) and are therefore regular.

∀L∈RL:∃p∈Z+:∀w∈L,∣w∣≥p:∃xyz=w,∣y∣>0:xy∗z⊆L \forall L \in \mathrm{RL} : \exists p \in \Z^+ : \forall w \in L, |w| \ge p : \exists xyz = w, |y| > 0 : xy^\ast z \subseteq L ∀L∈RL:∃p∈Z+:∀w∈L,∣w∣≥p:∃xyz=w,∣y∣>0:xy∗z⊆L

Also, ∣xy∣≤p|xy| \le p∣xy∣≤p.

Alternatively, as three properties:

  1. ∣y∣≥1|y| \ge 1∣y∣≥1
  2. ∣xy∣≤p|xy| \le p∣xy∣≤p
  3. ∀n≥0:xynz∈L\forall n \ge 0 : xy^n z \in L∀n≥0:xynz∈L

For L⊆Σ∗L \subseteq \Sigma^\astL⊆Σ∗,
∀p:∃s,∣s∣≥p:(∀xyz=s:¬P(x,y,z))  ⟹  L∉RL . \forall p : \exists s, |s| \ge p : \big( \forall xyz = s : \lnot P(x, y, z) \big) \implies L \not\in \mathrm{RL} \ . ∀p:∃s,∣s∣≥p:(∀xyz=s:¬P(x,y,z))⟹L∈RL .

where p∈Z+p \in \Z^+p∈Z+, s∈Ls \in Ls∈L, and PPP is the pumping lemma (the three properties).

Pushdown Automata

Nondeterministic PDAs (NPDA) are more powerful than Deterministic PDAs (DPDA).
NPDAs can recognize all Context-Free (Type-2) Languages, while DPDAs recognize only a subset.
Buuut we won’t talk about DPDAs :) .

A (nondeterministic) Pushdown Automaton ((N)PDA) is 6-tuple ⟨Q,Σ,Γ,δ,q0,F⟩\braket{Q, \Sigma, \Gamma, \delta, q_0, F}⟨Q,Σ,Γ,δ,q0​,F⟩:

  • Γ\GammaΓ – Stack Alphabet
  • δ\deltaδ – Transition Function
    δ:Q×Σε×Γε↦P(Q×Γε)\delta : Q \times \Sigma_\varepsilon \times \Gamma_\varepsilon \mapsto \mathcal{P}(Q \times \Gamma_\varepsilon)δ:Q×Σε​×Γε​↦P(Q×Γε​)
  • everything else – same as those of an NFA

Note that PDAs are FAs equipped with auxiliary storage in the form of a (LIFO) stack.
…And we’re smushing λ\lambdaλ's responsibility of outputting Γ\GammaΓ symbols into δ\deltaδ.1

In addition to being in a set of states, every point in a run also has an associated stack along its contents. Every transition depends on the top of the stack and manipulates it.

A δ(…,…,a)→(…,b)\delta(\dots, \dots, a) \to (\dots, b)δ(…,…,a)→(…,b) transition replaces the top of the stack (which must be aaa) with bbb.

ε\varepsilonεs are also allowed for Γ\GammaΓ in both δ\deltaδ's input and output, where ε→γ\varepsilon \to \gammaε→γ pushes γ\gammaγ onto the stack, γ→ε\gamma \to \varepsilonγ→ε pops (γ\gammaγ from) the stack, and ε→ε\varepsilon \to \varepsilonε→ε leaves the stack unchanged.
